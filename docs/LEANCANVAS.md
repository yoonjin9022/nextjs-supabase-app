## 항목별 상세 설명

### 1. 문제 (Problem)

유료 주차장을 이용하는 운전자가 겪는 4가지 핵심 불편:

| #   | 문제 상황                 | 현재 불편함                                                  |
| --- | ------------------------- | ------------------------------------------------------------ |
| 1   | **요금 불투명**           | 주차요금 안내판을 다시 찾아보거나 암산해야 함                |
| 2   | **최적 출차 타이밍 모름** | 10분 단위 요금인데 9분에 나가면 손해, 1분만 더 있으면 이득   |
| 3   | **예산 초과 인지 지연**   | 출차 후에야 요금이 예산을 초과했다는 걸 알게 됨              |
| 4   | **월 주차비 파악 어려움** | 한 달에 주차비를 얼마 썼는지 따로 기록하지 않으면 알 수 없음 |

---

### 2. 고객 세그먼트 (Customer Segments)

**주요 타겟**

- 일상적으로 유료 주차장을 이용하는 **개인 운전자**
- 주차비 지출을 의식하며 절약하고 싶은 운전자
- 스마트폰(모바일 브라우저)으로 빠르게 확인하려는 사용자

**초기 집중 타겟**

- 수도권 도심 직장인 (매일 유료 주차를 이용하는 반복 사용자)

---

### 3. 가치 제안 (Unique Value Proposition)

> **"지금 나가면 얼마?" 를 즉시 알 수 있게 한다**

- 입차 시간과 요금 체계 입력만으로 **현재 요금을 실시간으로** 확인
- 지금 / +5분 / +10분 / +30분 출차 요금 비교로 **최적 타이밍** 제안
- 예산 설정 → 소진 경보로 **불필요한 지출 방지**

---

### 4. 해결책 (Solution)

| 기능                                          | 해결하는 문제                  |
| --------------------------------------------- | ------------------------------ |
| **실시간 요금 계산기** (1초 갱신)             | 요금 불투명                    |
| **출차 타이밍 비교 카드** (지금/+5/+10/+30분) | 최적 출차 타이밍 모름          |
| **예산 Progress Bar + 경보**                  | 예산 초과 인지 지연            |
| **월별 주차 기록 & 통계**                     | 월 주차비 파악 어려움          |
| **즐겨찾기 주차장 관리**                      | 매번 요금 체계 재입력 번거로움 |

---

### 5. 경쟁우위 (Unfair Advantage)

- **국내 주차요금 체계 특화**: 기본요금 + 추가요금 단위 방식에 최적화된 계산 로직
- **모바일 퍼스트 단순 UX**: 주차장 입구에서 30초 이내 세션 시작 가능
- **개인화 데이터 영속성**: 로그인 기반으로 즐겨찾기·기록이 기기 변경 후에도 유지
- **오프라인 계산**: 요금 계산은 클라이언트에서 수행 → 네트워크 불안정 환경에서도 동작

---

### 6. 수익 흐름 (Revenue Stream)

| 단계 | 모델          | 내용                                                  |
| ---- | ------------- | ----------------------------------------------------- |
| MVP  | 무료          | 수익 모델 검증 전, 사용자 확보에 집중                 |
| 성장 | 프리미엄 구독 | 다중 차량 관리, 팀 공유, 고급 통계 (월 ₩1,990~₩4,990) |
| 확장 | B2B 제휴      | 주차장 사업자 대상 광고 노출 또는 API 제공            |

---

### 7. 비용 구조 (Cost Structure)

| 항목            | 세부 내용                                                 |
| --------------- | --------------------------------------------------------- |
| **인프라**      | Supabase (DB·인증·RLS), Vercel (배포·CDN)                 |
| **도메인**      | 연간 도메인 등록 비용                                     |
| **개발·운영**   | 솔로 개발자 시간 (기회비용)                               |
| **초기 고정비** | 매우 낮음 — Supabase Free tier + Vercel Hobby로 시작 가능 |

---

### 8. 주요 지표 (Key Metrics)

| 지표                       | 측정 기준                                      |
| -------------------------- | ---------------------------------------------- |
| **MAU** (월간 활성 사용자) | 월 1회 이상 세션 시작한 사용자 수              |
| **월 세션 시작 수**        | 전체 주차 세션 생성 횟수                       |
| **출차 완료율**            | 세션 시작 대비 출차 완료 비율 (이탈 방지 지표) |
| **즐겨찾기 등록 수**       | 재방문 의도 및 앱 고착도 지표                  |
| **재방문율**               | 7일 / 30일 내 재사용 비율                      |
| **예산 초과 방지율**       | 예산 설정 세션 중 예산 내 출차 비율            |

**Phase 1 목표 (MVP 출시 후 1개월)**

- 세션 시작 수 > 100건
- 출차 완료율 > 70%
- 재방문율 (7일) > 40%

---

### 9. 채널 (Channels)

| 채널                | 방식                                                   |
| ------------------- | ------------------------------------------------------ |
| **모바일 브라우저** | 반응형 웹 (PWA 미포함), 북마크 유도                    |
| **SNS 공유**        | 출차 완료 결과 카드 공유 기능                          |
| **커뮤니티**        | 자동차·주차 관련 온라인 커뮤니티 (보배드림, 클리앙 등) |
| **지인 추천**       | 입소문 (Word of Mouth)                                 |
